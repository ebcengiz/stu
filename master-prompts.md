# Master Prompt Koleksiyonu
# Tum projelerinizde kullanabileceginiz 10 uretim seviyesi prompt

---

## 1. Uretim Seviyesinde Ozellik Gelistirici (Production Feature Builder)

```
Uretime hazir ozellikleri gelistirmekten sorumlu kidemli (senior) bir yazilim muhendisi gibi davran. Amacin; temiz bir mimari ile olceklenebilir ve surdurulebilir bir ozellik tasarlamak ve gelistirmek.

Kod yazmaya baslamadan once:
- Gereksinimleri analiz et
- Edge case'leri (uc durumlari) belirle
- Mimariyi tanimla
- Uygulama planini olustur

Ardindan adim adim gelistir.

Ozellik: [Ozelligi tanimla]
Kullanicilar: [Hedef kullanicilar]
Teknoloji yigini: [Stack]
kisitlar: [Performans / SSR / minimum bagimlilik / vb.]

Ciktin sunlari icermeli:
- Mimari genel bakis
- Klasor yapisi
- Veri akisi
- Tam implementasyon (uygulama)
- Edge case (uc durum) yonetimi
- Hata yonetimi
- Performans degerlendirmeleri

Temiz, olceklenebilir ve bakimi kolay, uretim kalitesinde kod yaz.
```

## 2. Sifirdan Tam Uygulama (Full App From Scratch)

```
Uretime hazir, eksiksiz bir uygulama gelistiren kidemli full-stack muhendis gibi davran. Once sistem mimarisini tasarla, ardindan minimal ama olceklenebilir bir versiyon gelistir.

Kod yazmadan once sunlari dusun:
- Veritabani tasarimi
- API yapisi
- Arayuz (UI) mimarisi
- State yonetimi (durum yonetimi)

Uygulama fikri: [Acikla]
Temel ozellikler: [Listele]
Kullanicilar: [Hedef kullanicilar]
Teknoloji yigini: [Stack]

Sunlari dondur:
- Mimari
- Klasor yapisi
- Veritabani semasi
- API endpoint'leri
- UI yapisi
- Tam kod

Gercek bir startup MVP'si gibi tasarla ve olceklenebilir olsun.
```

## 3. Kod Tabanini Anlama + Refactor

```
Buyuk ve yabanci bir kod tabanina dahil olan kidemli muhendis gibi davran. Once mimariyi ve veri akisini anla.

Sonra sunlari tespit et:
- Yapisal problemler
- Tekrar eden kodlar
- Performans darbogazlari
- Surdurulebilirlik riskleri

Ardindan iyilestirme oner ve refactor et.

Sunlari dondur:
- Mimari ozeti
- Problemli alanlar
- Refactor stratejisi
- Gelistirilmis mimari
- Yeniden yazilmis kod

Fonksiyonellik ayni kalsin, kalite artsin.
```

```
Edge case'leri ve performansi da dusun.

Kod:
[Buraya yapistir]

- Neden hata veriyor
- Edge case'ler
- Duzeltilmis production-ready kod
```

## 8. Coklu Agent Workflow

```
Sen birlikte calisan 4 agentsin:

- Architect (mimar) -> sistemi tasarlar
- Engineer (gelistirici) -> gelistirir
- Reviewer (denetleyici) -> kaliteyi kontrol eder
- Optimizer (optimizasyoncu) -> performansi iyilestirir

Her agent kendi rolunu yerine getirsin ve ciktisini bir sonrakine devretsin.

Gorev:
[Acikla]

Sunlari dondur:
- Mimari (Architect ciktisi)
- Implementasyon (Engineer ciktisi)
- Review feedback (Reviewer ciktisi)
- Optimize edilmis final versiyon (Optimizer ciktisi)
```

# Kullanim Rehberi

Yukaridaki promptlardan ihtiyaciniza uygun olani secin, koseli parantez `[...]` icindeki alanlari kendi projenize gore doldurun ve Claude'a gonderin.

Birden fazla promptu birlestirebilirsiniz. Ornegin:
- Yeni proje basliyorsaniz: **2 + 5** (Full App + Sistem Tasarimi)
- Mevcut kodu iyilestiriyorsaniz: **3 + 6 + 7** (Refactor + Performans + Temiz Mimari)
- Bug cozuyorsaniz: **4** (Debugging)
- Yeni ozellik ekliyorsaniz: **1 + 9 + 10** (Feature + UI + API)
